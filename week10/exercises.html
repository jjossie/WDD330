Week 10 Notes

Client-Side Form Validation

  • Client-Side validation happens to provide a good user experience, but doesn't guarantee that there won't ever be bad data passed to the server. For that you need server-side validation.
  • Inputs that are valid based on HTML built-in validation like min/max and type automatically have the css :valid pseudoclass or :invalid if not.
  • pattern lets you match a regular expression to validate.
  • The Constraint Validation API lets you access the details of HTML validation from javascript.
  • It provides objects like HTMLInputElement which has a validity property which is a ValidityState object with properties like tooLong or rangeOverflow.

Using the Fetch API

  • fetch will resolve every time it gets an HTTPResponse back. Even if it's an error like 404. the status.ok/ will be false, but it won't default to the .catch on the promise unless you throw the error yourself.
